```
2. Journey — a vertical scroll-driven timeline through the JOURNEY entries in
   content.js. This is the section that carries the site's signature
   interaction, build it carefully:
   - An SVG path/line connects each timeline card in sequence, drawn using
     stroke-dasharray/dashoffset (or a Motion Path), controlled by a GSAP
     ScrollTrigger with `scrub: true` mapped to the section's scroll progress.
   - Because scrub is tied directly to scroll position, this is naturally
     bidirectional: scrolling down draws the line forward, scrolling up
     retracts it — no extra logic needed for the reverse case, that's what
     scrub gives you for free. Do NOT use toggleActions/one-shot triggers
     for this line.
   - Make the line's motion clearly visible, not a faint accent — it should
     read as an obvious draw/retract, not subtle background decoration.
   - Individual cards can have a lighter fade + y-translate reveal as they
     enter (this part CAN be a simpler one-shot animation for performance),
     alternating left/right on desktop, single column on mobile.

Mobile-first: build the single-column mobile layout first, then add the
desktop enhancements at md: breakpoints and above. Test at 375px width. On
mobile, keep the scrub-linked line but simplify/shorten its path if needed
for performance on low-end phones.
```
